Q2 of 19

You are the Technical Lead of your project, and your project code is hosted on a CodeCommit repository. To be constantly updated with what code changes are pushed to the repository, you want to receive an email with every code change a developer pushes. From the given options, which one is the best way to set this up?

Options

1. Configure a CloudWatch Events rule to send a message to SES.
SES will then trigger an email sent whenever a user pushes code to the repository.

2. Create a new SNS topic and configure it to poll for CodeCommit events at regular intervals.
You can subscribe to the topic to receive email notifications.

3. Configure Notifications using the CodeCommit console. This will create a CloudWatch Events rule which sends notifications to an SNS topic.
You can subscribe to the topic to receive email notifications.

4. Configure a CloudWatch Events rule to send a message to SQS.
SQS will then trigger an email sent whenever a user pushes code to the repository.
