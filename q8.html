Q8 of 19

You have an application hosted on AWS created using CloudFormation Templates and Autoscaling. Your application gets a sudden increase in traffic, and this is affecting the application performance. A change of instance type to C5 would resolve the issue. How can this be implemented while minimizing downtime for end users?

Options

1. Update the launch configuration in the AWS CloudFormation template with the new C5 instance type.
Add an UpdatePolicy attribute to your Auto Scaling group that specifies an AutoScalingRolling Update in order to avoid downtime.
The instances will get automatically updated.

2. Update the launch configuration in the AWS CloudFormation template with the new C5 instance type.
Run a stack update with the updated template and Auto Scaling will then update the instances one at a time with the new instance type.

3. Update the launch configuration in the AWS CloudFormation template with the new C5 instance type.
Add an UpdatePolicy attribute to the Auto Scaling group that specifies an AutoScalingRollingUpdate.
Run a stack update with the updated template.

4. Copy the old launch configuration of the template and create a new launch configuration with the C5 instances.
Update the Auto Scaling group with the new launch configuration.
Auto Scaling will then update the instance type of all running instances.
